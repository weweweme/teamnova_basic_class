## ACT 1 · GET은 주소에 다 드러난다 (1분 30초)

| 동작 | 주소 | 말할 것 |
|---|---|---|
| **작품** 클릭 | `/works.php` | |
| **드라마** 필터 | `/works.php?genre=드라마` | "**필터가 주소에 붙었죠.** 이 주소를 친구에게 보내면 똑같은 화면이 뜹니다" |
| **검색** → `놀란` | `/search.php?q=놀란` | "인터스텔라, 인셉션 2편. **감독으로도 찾습니다**" |

**검색어에 노란 형광펜이 칠해진 것 짚기**

> "검색어를 강조하려면 **먼저 안전 처리하고, 그 다음에 태그를 넣어야** 합니다.
> 순서를 바꾸면 태그가 글자로 보이거나, 더 나쁘면 **XSS 구멍**이 생겨요."

> 💡 **"GET은 투명한 봉투예요. 뭘 요청했는지 주소에 다 적혀 있죠."**

## ACT 2 · ★ 하이라이트: 파라미터 5개 겹치기 (2분)

**기생충** 클릭 → `/board/?work=parasite` (총 7개 · 3페이지)

**탭을 하나씩 누르며 주소가 자라는 걸 보여주기:**

```
?work=parasite
?work=parasite&sort=hot                          ← 인기 탭
?work=parasite&sort=hot&sentiment=호평           ← 호평 탭
?work=parasite&sort=hot&sentiment=호평&page=2    ← 2페이지
?work=parasite&q=음악&sort=hot&sentiment=호평    ← 검색
```

> **핵심 대사:** "탭을 눌러도 **앞의 조건이 안 풀립니다.**
> `query_url()`이 나머지를 유지하고 하나만 바꾸거든요."

> **함정 소개:** "GET 폼은 제출하면 **기존 파라미터가 다 날아갑니다.**
> 그래서 검색창에 작품을 `hidden`으로 몰래 실어 보내요.
> 안 그러면 검색하는 순간 작품이 풀립니다."

**작품 투표 토글:**
👍 추천(빨갛게 채워짐) → 👍 **또** 누름(해제) → 👎 비추천(**총 표 그대로**)

> "표가 **옮겨갈 뿐 늘지 않습니다.** 1인 1표니까요."

## ACT 3 · POST와 PRG (2분 30초)

### ① 비로그인 방어
글쓰기 클릭 → 로그인 페이지 + 🔒 **우상단 토스트**

> "화면에서 버튼을 숨기는 건 편의고, **서버가 진짜로 막습니다.**"

### ② 로그인 (`영화광` / `1234`)
👋 환영 토스트 → **3초 뒤 스르륵 사라짐**

> "로그인 정보는 **주소에 안 보이죠.** POST는 봉투 안에 넣어 보내니까요.
> 비밀번호가 주소에 남으면 큰일입니다."

### ③ 글쓰기
- 제목 타이핑 → **`12 / 100` 카운터가 실시간으로 올라감**

> "화면에서 미리 알려주지만, **서버에서도 다시 셉니다.**
> 화면 검사는 친절이고, **서버 검사가 진짜 방어**예요."

- 내용 쓰다가 **뒤로가기** → "작성 중인 내용이 사라집니다" 경고 → 취소
- 등록 → 주소 확인

```
POST /post/create.php  →  [302]  →  GET /board/?work=inception
```

- **여기서 F5 누르기**

> "새로고침해도 **글이 또 안 써집니다.** 이게 PRG예요.
> 주소창에 남은 건 GET 주소니까 조회만 다시 일어나죠.
> 리다이렉트를 빼먹으면 쇼핑몰에서 **결제가 두 번 되는 사고**가 납니다."

### ④ 추천 토글
👍 클릭(파랗게 채워짐) → 다시 클릭(취소)

> "**1인 1회**입니다. '몇 번 눌렀나'가 아니라 '눌렀나 안 눌렀나'를 저장해요."

### ⑤ 댓글
댓글 작성 → 목록으로 나가서 **댓글 수가 같이 오른 것** 보여주기

> "댓글 수를 따로 저장 안 하고 **실제 댓글을 셉니다.**
> 두 군데 저장하면 반드시 어긋나거든요."

## ACT 4 · ★ 삭제 → 되돌리기 (1분)

내 글에서 **🗑 삭제** 클릭

**`<dialog>` 확인 모달**이 뜸 (브라우저 기본 창 아님)

> "신고 팝업과 **같은 방식으로 통일**했습니다."

확인 → 게시판으로 돌아옴 + **토스트에 `[되돌리기]` 버튼**

**되돌리기 클릭** → **글이 그대로 복구**

```
POST /post/restore.php  →  [302]  →  GET /post/view.php?id=…
```

> **핵심 대사:** "삭제를 '진짜 지우기'가 아니라 **'지웠다고 표시하기'**로 만들면
> 되돌리기가 공짜로 생깁니다.
> 실무 DB에서도 `DELETE` 대신 `deleted_at` 칼럼을 두는 **소프트 삭제**를 쓰는 이유예요."

> 덧붙이기: "되돌리기도 **상태를 바꾸는 동작이라 POST**입니다. 링크가 아니에요."

## ACT 5 · 보안 (1분 30초) — **점수 나는 구간**

### ① 소유권
- 내 글 → 수정·삭제 버튼 **있음**
- 남의 글 → 버튼 **없음**

### ② F12 콘솔에서 강제 삭제 시도 ← 하이라이트

```js
fetch('/post/delete.php', {method:'POST', body:new URLSearchParams({id:2})})
  .then(r => console.log(r.url))
// → /post/view.php?id=2   (안 지워짐)
```

새로고침 → 🔒 **"본인이 쓴 글만 수정·삭제할 수 있습니다"**

> "버튼을 숨겨도 요청은 조작할 수 있습니다. **그래서 서버에서 다시 확인해요.**
> **화면 숨김 = 편의, 서버 확인 = 진짜 보안**입니다."

### ③ 주소로 삭제 시도
`localhost:8081/post/delete.php?id=1` → 홈으로 튕김

> "삭제를 GET 링크로 만들면 **검색엔진 크롤러가 훑기만 해도 글이 다 지워집니다.**
> 실제로 있었던 사고예요."

### ④ 이상한 입력
- `/board/?page=-99` → 1페이지로 보정
- `/board/?work[]=x` → 멀쩡 (배열이 와도 안 터짐)

홈으로 이동 → **"최근 본 글"** 목록 짚기

> "방금 본 글들이 쌓여 있죠. 이건 GET인데도 세션에 기록합니다.

## 예상 질문 대비

| 질문 | 답 |
|---|---|
| "왜 GET/POST를 나눴나?" | GET은 조회(주소 공유 가능), POST는 변경. 삭제를 GET으로 두면 크롤러가 지웁니다 |
| "PRG가 뭔가?" | POST 처리 후 리다이렉트. 새로고침해도 중복 제출이 안 됩니다 |
| "세션이 뭔가?" | 서버가 '이 브라우저 = 이 사용자'를 기억하는 저장공간. 브라우저엔 번호표(쿠키)만 줍니다 |
| "비밀번호는?" | `password_hash()`로 저장, 되돌릴 수 없습니다. 평문 저장 안 합니다 |
| "DB는 왜 없나?" | 다음 주차 범위입니다. `includes/` 함수 속만 SQL로 바꾸면 됩니다 |
| "화면에서 버튼 숨겼는데 왜 서버에서 또 검사?" | 요청은 조작할 수 있으니까요 (ACT 5 ②에서 직접 보여드림) |
